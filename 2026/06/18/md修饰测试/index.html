Hexo博客搭建与部署踩坑全记录（md修饰）

发表于2026-06-18|技术笔记

（以下内容由deepseek根据对话整理）这是一篇记录我从零开始搭建 Hexo 博客，并成功部署到 GitHub Pages 的完整过程。期间遇到了主题安装、脚本报错、配置文件格式、网络连接、SSH 权限等一系列问题，最终全部解决。希望这份记录能帮助到遇到类似问题的朋友。

背景

我最初的目标是搭建一个类似 Hatoya 的个人博客，用于发布学习笔记。选择了基于 Node.js 的静态博客框架 Hexo，并使用 GitHub Pages 进行免费托管。

环境与工具

  • 操作系统：Windows
  • 博客框架：Hexo
  • 主题：bcxm（基于 Butterfly 的修改版）
  • 部署平台：GitHub Pages
  • 版本控制：Git + SSH

5. SSH 密钥权限为只读（marked as read only）

问题描述：

在解决网络问题后，部署时出现新错误：

ERROR: The key you are authenticating with has been marked as read only.
fatal: Could not read from remote repository.

原因分析：

我添加到 GitHub 的 SSH 公钥，在创建时被赋予了”只读”权限，无法执行推送（写入）操作。

解决方案：

第一步，登录 GitHub → Settings → SSH and GPG keys。

第二步，删除原有的公钥记录。

第三步，重新点击 New SSH Key，在添加时确保 “Allow write access” 选项被勾选。

第四步，重新执行 hexo deploy，部署成功。

总结与心得

第一，遇到错误，先看日志。 Hexo 和 Git 的错误信息都非常具体，是定位问题的最佳线索。

第二，配置文件的格式是基础。 YAML 的缩进和语法错误是新手最容易遇到的问题，使用专业的代码编辑器（如 VS Code）并安装语法高亮插件能有效避免。

第三，网络问题是部署的最大拦路虎。 GitHub 在国内的访问稳定性不一，提前配置好 SSH 代理或熟悉 SSH over HTTPS 端口的技巧能节省大量时间。

第四，权限管理要细心。 在 GitHub 上设置 SSH 密钥或 Token 时，务必确认赋予了正确的操作权限（读写）。
